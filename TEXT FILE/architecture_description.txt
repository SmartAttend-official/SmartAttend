OPTION 3: DETAILED TEXT SPECIFICATION (For manual recreation or AI reading)
--------------------------------------------------------------------------------
### 1. FRONTEND ARCHITECTURE (Client-Side)
- **Technology**: Vanilla HTML5, CSS3 (Glassmorphism theme, dark mode preferred), and Plain JavaScript.
- **Key Modules**:
    - `Login.html` / `login.js`: Handles multi-role login (Admin/Professor/Student) and session token validation.
    - `Record.html` / `record.js`: Interface for viewing and exporting attendance history (uses jsPDF and SheetJS).
    - `Start Attendance.html` / `start_attendance.js`: Interface for professors to mark attendance in real-time.
    - `Leave_Requests.html` / `leave_requests.js`: Interface for students to submit leaves and professors to approve them.

### 2. BACKEND ARCHITECTURE (Cloud API)
- **Technology**: Google Apps Script (GAS).
- **Core File**: `SmartAttend_AppScript_FINAL.gs` (approx. 2,400+ lines).
- **Key API Actions (GET)**:
    - `verify_otp`: Validates 5-digit recovery codes.
    - `subject_stats`: Fetches live attendance percentage calculated from raw history.
    - `predict`: Calculates how many classes a student can miss or must attend to reach 75%.
- **Key API Actions (POST)**:
    - `verify_login`: Validates credentials and returns a UUID Session Token.
    - `save_session`: Records a class session, updates student histories, and triggers background calculations.

### 3. DATABASE ARCHITECTURE (Storage)
- **Technology**: Google Sheets (Multi-Sheet distributed architecture).
- **Spreadsheets**:
    - **Student Sheet**: Contains `Sheet2` (Master student records with `Classes_Attended`, `Total_Classes`, and `Attendance_Percentage`) and `AttendanceHistory` (Every single present/absent log).
    - **Professor Sheet**: Contains professor credentials, assigned departments, and `AuditLogs` for security tracking.
    - **Academic Sheet**: Contains registries for `Departments`, `Classes`, `Subjects`, and `Proxies`.

### 4. WORKFLOWS
- **Attendance Flow**: Professor submits list -> Apps Script appends to `AttendanceHistory` -> Apps Script recalculates `Sheet2` totals -> Apps Script emails summary to Professor and alerts to Parents of absent students.
- **Leave Flow**: Student submits request -> Apps Script logs to `LeaveRequests` -> Professor approves -> Apps Script updates status and notifies Parent.
