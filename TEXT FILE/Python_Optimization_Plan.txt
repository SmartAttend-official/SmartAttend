=====================================================
SMARTATTEND - FUTURE PYTHON OPTIMIZATION PLAN
=====================================================
Context: As the system scales beyond a single student to a full class or university, the Python face recognition scripts will need optimization to prevent high CPU usage, slow response times, and memory crashes.

1. LIVE WEBCAM OPTIMIZATIONS (test_live_webcam.py)
-----------------------------------------------------
* Frame Resizing: The most effective way to speed up face recognition is to shrink the video frame (e.g., to 1/4 size) before passing it to the face_recognition library. The detections are scaled back up for the display box.
* Frame Skipping: Instead of processing all 30 frames per second, the script should process every 3rd or 5th frame for face matching, while simply continuing to draw the bounding boxes on the intermediate frames.

2. ENCODING GENERATION OPTIMIZATIONS (generate_encoding.py)
-----------------------------------------------------
* Batch Processing: When generating encodings for hundreds of students, load and process images in batches (e.g., 50 at a time) to prevent RAM overload.
* Error Handling: Add checks to skip images where no face is detected, logging the error rather than crashing the loop.

3. MEMORY & ALGORITHMIC EFFICIENCY
-----------------------------------------------------
* NumPy Vectorization: Ensure face distance calculations leverage NumPy's vectorized operations instead of standard Python loops for much faster matching.
* Pickle Caching: Load the generated encodings (`encodings.pickle`) once when the webcam script starts, and keep it in memory rather than reloading it.

Note: These optimizations will become critical when moving from testing with 1 student to testing with 10+ students.
